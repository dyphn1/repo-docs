# AGENTS.md Template

Terse, directive. Every section should be machine-readable.
No filler. No prose padding. Commands must be exact and copy-pasteable.

---

## Archetype-Aware Structure

The sections to include depend on the repository archetype (from Step 0 of the skill
workflow). For `code` repos, use the full template below. For other archetypes, replace
the **Commands** block with **Workflows & Commands** and use the relevant non-code
template at the end of this file.

| Archetype | Commands block | Extra sections |
|---|---|---|
| **`code`** | Full `## Commands` with setup/dev/test/lint/build | Architecture, Conventions, Do Not Edit, Gotchas |
| **`product`** | `## Development Commands` (Setup, Dev, Test, Build) | Tech Stack, Domain Knowledge Mapping, System Boundaries |
| **`single-skill`**| `## Commands` (Testing, Linting) | See [`skills-agents-template.md`](skills-agents-template.md) (Taxonomy, Extension Rules, Schema) |
| **`multi-skills`**| `## Commands` (Authoring, Linking, Linting) | See [`skills-agents-template.md`](skills-agents-template.md) (Taxonomy, Extension Rules, Schema) |
| **`docs/courseware`**| `## Editing Rules & Formatting` | See [`knowledge-agents-template.md`](knowledge-agents-template.md) (Agentic Workflows, Task States) |
| **`docs`** | `## Workflows & Commands` (build, preview, publish) | Page Tree, Authoring Rules, Style Conventions |
| **`hybrid`** | Combine applicable blocks | Combine relevant extra sections |

```

### Lint exercises
```bash
{lint command, e.g., pnpm ai-hero-cli internal lint}
```

### Commit with history preservation
```bash
git mv exercises/{old-path} exercises/{new-path}  # use git mv, never plain mv
```

## Exercise Structure

```
exercises/
  {XX}-{section-name}/
    {XX}.{YY}-{exercise-name}/
      explainer/   # conceptual material — no TODOs
      problem/     # student workspace with TODOs
      solution/    # reference implementation
```

## Naming Conventions

- Sections: `XX-section-name/` (two-digit zero-padded prefix, dash-case)
- Exercises: `XX.YY-exercise-name/` (section.exercise prefix, dash-case)
- Each subfolder must contain a non-empty `readme.md`

## Lint Rules Summary

- Each exercise needs at least one of `problem/`, `explainer/`, or `solution/`
- `readme.md` must exist and be non-empty in each subfolder
- No `.gitkeep` files; no broken links in readmes

## Style rules for AGENTS.md

- **Imperative** sentences: "Run X to do Y" not "You can run X..."
- **No intros or sign-offs** — agents don't need greetings
- All commands must be runnable as-is in a terminal (no pseudocode)
- If a command varies by OS, show both: `# macOS / Linux` and `# Windows`
- Be explicit about what NOT to do — agents benefit from guardrails
- List env vars agents will need; never embed secrets
